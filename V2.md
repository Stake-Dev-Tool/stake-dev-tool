# Stake Dev Tool V2 — Cloud platform

Working plan for V2. This branch (`v2`) is where the cloud platform is built.
Everything here is open source; the paid offering is **our hosted instance**,
nothing else. Self-hosters get 100% of the features.

## Vision

V1 is a desktop workbench with cloud features bolted on (GitHub-repo teams,
GitHub Pages share). V2 inverts this: **one engine (the `lgs` crate), three
surfaces**:

1. **Desktop app** — local dev loop (front hot-reload, local math), unchanged
   core role.
2. **Web workbench** — the test view running against the cloud server. Math
   devs, QA and PMs need zero install.
3. **Share links** — real hosted game instances on `<slug>.play.<domain>`,
   backed by a real server-side RGS.

### Product model (open-core, Plausible-style)

- The entire platform is open source and fully self-hostable
  (`docker compose up`).
- The subscription sells **hosting**: zero-install, wildcard subdomains,
  storage, backups, updates. No feature gating, no enterprise edition.
- Billing code only runs on the hosted instance; self-hosters never see it.

## Architecture

```
                    ┌─ Caddy (TLS, wildcard *.play.<domain>) ─┐
                    │                                          │
   app.<domain> ────┤                                          │
   *.play.<domain> ─┤          crates/server (axum)            │
                    │  ┌────────────┬──────────────┬────────┐  │
                    │  │ API + auth │ multi-tenant │ share  │  │
                    │  │ + sync     │ LGS (lgs)    │ router │  │
                    │  └─────┬──────┴──────┬───────┴───┬────┘  │
                    └────────┼─────────────┼───────────┼───────┘
                             │             │           │
                        Postgres      blob cache   object store
                        (sqlx)        (mmap LRU)   (fs / S3-R2)
```

- **`crates/server`** — new axum binary. Embeds the dashboard + workbench UI
  (SvelteKit `adapter-static`, same pattern as the LGS serving the test view
  today). Single static binary; self-host = binary (or Docker image) +
  Postgres + Caddy.
- **`crates/protocol`** — shared request/response types, TS generated via
  `ts-rs` (one source of truth for desktop, CLI, dashboard, server).
- **`crates/lgs`** — unchanged role, embedded by both desktop and server.
  Gains multi-tenant session scoping.
- **`crates/cli`** — `stake-dev-tool push` etc., for CI pipelines (math is
  generated by simulations; the real workflow is "math CI pushes a
  revision").
- **`ui/`** — factored so the test view can be served by desktop *or* cloud.
- **`crates/lgs-wasm`** — removed from the critical path (share no longer
  uses it). Kept only if a "static offline export" feature is ever wanted;
  otherwise deprecate.

### Storage

- **Postgres** (sqlx, versioned migrations) — users, workspaces, revisions,
  documents, share links, sessions metadata.
- **Object store** via the `object_store` crate — local filesystem for
  self-host, S3-compatible (R2) for the hosted instance. Holds math blobs and
  front bundles.

## Core designs

### Auth

- Email + password (argon2) and GitHub OAuth. Cookie sessions for the web.
- **Desktop**: OAuth device flow (or loopback redirect) → long-lived token.
- **CI**: personal/workspace API tokens with scopes (e.g. `push:math`).
- No external identity service — self-host must work offline from any vendor.

### Workspaces

`users` → `workspaces` → `memberships (role: owner | admin | member)`.
Games belong to a workspace. All quotas hang off the workspace.

### Math revisions (replaces GitHub-repo team sync for math)

Immutable snapshots + content-addressed blobs. **Not** git.

- `blobs`: content hash → object store key, size, refcount. A
  `books_*.jsonl.zst` unchanged between rev 41 and 42 is never re-uploaded
  nor re-stored (dedup by hash). Chunked, resumable uploads.
- `math_revisions`: id, workspace, game, author, message, parent_rev,
  manifest (path → blob hash), created_at.
- Push flow: client sends manifest → server answers with missing hashes →
  client uploads only those → server commits the revision atomically.
- Broadcast via workspace SSE stream: "rev 42 pushed by X". Active LGS
  sessions never hot-swap; new sessions take latest unless pinned.
- **Saved rounds reference `(revision, mode, eventId)`** → replays stay valid
  forever (fixes a structural V1 bug where a math change broke bookmarks).
- Server computes per-revision bet-stats (reusing `lgs`) → auto changelog
  between revs: RTP per mode, max win, modes added/removed.

### Document sync (profiles, saved rounds, bookmarks)

Server-authoritative versioned JSON documents: `revision + updated_at`,
last-write-wins, conflicts surfaced to the UI. Realtime via SSE (already the
app's idiom). This fully replaces the GitHub-repo teams system.

### Multi-tenant LGS

- Math roots materialized from object store into a local cache, keyed by
  `(workspace, game, revision)`.
- The existing mmap books cache with LRU eviction is reused, with per-
  workspace bounds so one big game can't starve others.
- Sessions are scoped: authenticated (workbench) or anonymous visitor
  (share), always pinned to a revision.

### Share v2 (real RGS — replaces the WASM/GitHub Pages share)

```
https://<slug>.play.<domain>
├── /            → game front bundle (static, from object store)
└── /api/rgs/... → multi-tenant LGS, workspace math at the pinned revision
```

- Same-origin front + RGS → no CORS, and the game talks to the RGS exactly
  like it will talk to Stake Engine in production (real wallet flow, real
  latencies, real event stream).
- **Math never leaves the server** → the Sampled/Partial/Full privacy modes
  disappear entirely (full fidelity *and* full privacy).
- Link options: pin a fixed revision or track latest; expiry; password.
- Replay links: share an exact `(revision, mode, eventId)` round.
- Visitor sessions: anonymous, ephemeral token, fake balance, TTL.
- Abuse/cost control: rate limits per IP and per link, concurrent-session
  caps per plan, link expiry.
- Analytics per link: sessions, spins, observed RTP. Feedback widget tied to
  the exact bet event (stretch).
- **Origin isolation**: `*.play.<domain>` is strictly separated from
  `app.<domain>` so dashboard cookies can never leak into shared games.

## Licensing

- Desktop app + `lgs` + `lgs-wasm`: stay **MIT**.
- `crates/server` (+ `protocol`, `cli`): **AGPL-3.0** — self-hosting is
  untouched (the DNA is preserved), but nobody can resell our own server as
  a closed hosted service. Same choice as Plausible / Cal.com.
- ⚠️ Decide **before** merging external contributions to the server.

## Hosting (our instance) & running costs

Boring and cheap on purpose. Rust keeps the footprint tiny; the dominant
cost driver is egress for front bundles/assets → R2 (zero egress fees).

| Item | Choice | Cost |
| --- | --- | --- |
| Compute | Hetzner Cloud VPS (CX22/CAX21), Docker + Caddy | ~€4–7/mo |
| Database | Postgres on the same VPS, dumps shipped to R2 | €0 |
| Object storage | Cloudflare R2 (10 GB free, then ~$0.015/GB/mo, **egress free**) | ~€0 to start |
| DNS | Cloudflare free plan | €0 |
| Wildcard TLS | `*.play.<domain>` in **DNS-only** mode + Caddy DNS-01 (Cloudflare API token). Do **not** proxy the wildcard (second-level wildcards need the paid cert plan) | €0 |
| Transactional email | Resend free tier (invites, signup) | €0 |
| Monitoring/analytics | PostHog EU free tier + UptimeRobot free | €0 |
| VPS backups | Hetzner auto-backup (+20% of VPS) | ~€1/mo |

**Fixed total: < €10/mo.** Marginal cost per customer ≈ storage only
(10 GB on R2 ≈ $0.15/mo). Two or three subscribers cover the infra.

Scaling path (later, only if needed): bigger VPS → second VPS + Hetzner LB.
No Kubernetes, no serverless.

## Pricing (draft — decision pending)

Goal: cheapest sustainable subscription. Marginal cost per workspace is
genuinely tiny, so low flat prices work if quotas cap the whales:

- **Self-host — free forever.** All features. This is the DNA.
- **Solo — ~€5/mo** (or ~€48/yr): 1 user, N games, 10 GB math storage,
  share links with fair-use session quotas.
- **Team — ~€15/mo** (or ~€144/yr): up to 10 members, 50 GB, higher share
  quotas, custom play subdomain.
- No permanent free cloud tier (it's the only thing that can cost us real
  money) — 14-day trial instead.
- Quotas are metered where cost lives: storage GB and concurrent share
  sessions. Overage as cheap add-ons (e.g. +10 GB for ~€1/mo).

Payment processing: at €5/mo, Stripe's fixed fee eats ~7–10% and **EU VAT
on cross-border B2C digital sales is our problem**. **Decision (2026-07):
Polar (polar.sh) as merchant of record** (4% + 40¢) — they remit VAT for
us; the site's billing module targets their API. Lemon Squeezy was ruled
out (in maintenance mode since the Stripe acquisition); Paddle is the
fallback if Polar's compliance review balks at the gambling-adjacent
niche — flag upfront that this is B2B dev tooling, not gambling. Push
annual billing (2 months free) to dilute fixed fees. Revisit raw Stripe
once revenue justifies the VAT admin.
